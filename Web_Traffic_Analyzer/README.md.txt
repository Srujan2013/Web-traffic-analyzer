 Mini Web Traffic Analyzer & Security Tester

 Overview
This project focuses on analyzing web traffic and understanding how client-server communication works using browser developer tools.

It also explores basic web security concepts like data transmission, request payloads, and common vulnerabilities.

 Objectives
- Understand HTTP/HTTPS communication
- Analyze GET and POST requests
- Inspect request/response headers
- Study login request payloads
- Explore basic security concepts

 Tools Used
 Google Chrome DevTools
 Command Prompt (ping, nslookup)


 Features
- Analyzed network requests in real-time
- Identified request methods (GET/POST)
- Observed status codes (200, 404, etc.)
- Inspected login form data (payload)
- Performed DNS and IP analysis


 Security Concepts Explored
- Difference between HTTP and HTTPS
- SQL Injection basics
- Cross-Site Scripting (XSS)
- Importance of secure data transmission

 What I Learned
- How web communication works internally
- How data is transmitted between client and server
- Basic understanding of web vulnerabilities
- Practical exposure to debugging tools

 Disclaimer
This project was conducted for educational purposes only on safe/demo environments.

 Future Improvements
